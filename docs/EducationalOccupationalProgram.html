EducationalOccupationalProgram implement Intangible

object

A program offered by an institution which determines the learning progress to achieve an outcome, usually a credential like a degree or certificate. This would define a discrete set of opportunities (e.g., job, courses) that together constitute a program with a clear start, end, set of requirements, and transition to a new occupational opportunity (e.g., a job), or sometimes a higher educational opportunity (e.g., an advanced degree).

Properties

programPrerequisites (inherited from EducationalOccupationalProgram)

Type AlignmentObject,Course,EducationalOccupationalCredential,string

Prerequisites for enrolling in the program.

hasCourse (inherited from EducationalOccupationalProgram)

Type Course

A course or class that is one of the learning opportunities that constitute an educational / occupational program. No information is implied about whether the course is mandatory or optional; no guarantee is implied about whether the course will be available to everyone on the program.

startDate (inherited from EducationalOccupationalProgram)

Type dateTime,date

The start date and time of the item (in [ISO 8601 date format](http://en.wikipedia.org/wiki/ISO_8601)).

salaryUponCompletion (inherited from EducationalOccupationalProgram)

Type MonetaryAmountDistribution

The expected salary upon completing the training.

termsPerYear (inherited from EducationalOccupationalProgram)

Type number

The number of times terms of study are offered per year. Semesters and quarters are common units for term. For example, if the student can only take 2 semesters for the program in one year, then termsPerYear should be 2.

dayOfWeek (inherited from EducationalOccupationalProgram)

Type DayOfWeek

The day of the week for which these opening hours are valid.

offers (inherited from EducationalOccupationalProgram)

Type Offer,Demand

An offer to provide this item—for example, an offer to sell a product, rent the DVD of a movie, perform a service, or give away tickets to an event. Use [[businessFunction]] to indicate the kind of transaction offered, i.e. sell, lease, etc. This property can also be used to describe a [[Demand]]. While this property is listed as expected on a number of common types, it can be used in others. In that case, using a second type, such as Product or a subtype of Product, can clarify the nature of the offer.

typicalCreditsPerTerm (inherited from EducationalOccupationalProgram)

Type number,StructuredValue

The number of credits or units a full-time student would be expected to take in 1 term however 'term' is defined by the institution.

applicationStartDate (inherited from EducationalOccupationalProgram)

Type date

The date at which the program begins collecting applications for the next enrollment cycle.

timeOfDay (inherited from EducationalOccupationalProgram)

Type string

The time of day the program normally runs. For example, "evenings".

endDate (inherited from EducationalOccupationalProgram)

Type date,dateTime

The end date and time of the item (in [ISO 8601 date format](http://en.wikipedia.org/wiki/ISO_8601)).

provider (inherited from EducationalOccupationalProgram)

Type Organization,Person

The service provider, service operator, or service performer; the goods producer. Another party (a seller) may offer those services or goods on behalf of the provider. A provider may also serve as the seller.

trainingSalary

Type MonetaryAmountDistribution

The estimated salary earned while in the program.

educationalCredentialAwarded (inherited from EducationalOccupationalProgram)

Type string,EducationalOccupationalCredential,string

A description of the qualification, award, certificate, diploma or other educational credential awarded as a consequence of successful completion of this course or program.

numberOfCredits (inherited from EducationalOccupationalProgram)

Type number,StructuredValue

The number of credits or units awarded by a Course or required to complete an EducationalOccupationalProgram.

timeToComplete (inherited from EducationalOccupationalProgram)

Type Duration

The expected length of time to complete the program if attending full-time.

occupationalCredentialAwarded (inherited from EducationalOccupationalProgram)

Type EducationalOccupationalCredential,string,string

A description of the qualification, award, certificate, diploma or other occupational credential awarded as a consequence of successful completion of this course or program.

termDuration (inherited from EducationalOccupationalProgram)

Type Duration

The amount of time in a term as defined by the institution. A term is a length of time where students take one or more classes. Semesters and quarters are common units for term.

educationalProgramMode (inherited from EducationalOccupationalProgram)

Type string,string

Similar to courseMode, The medium or means of delivery of the program as a whole. The value may either be a text label (e.g. "online", "onsite" or "blended"; "synchronous" or "asynchronous"; "full-time" or "part-time") or a URL reference to a term from a controlled vocabulary (e.g. https://ceds.ed.gov/element/001311#Asynchronous ).

programType (inherited from EducationalOccupationalProgram)

Type string,DefinedTerm

The type of educational or occupational program. For example, classroom, internship, alternance, etc..

maximumEnrollment (inherited from EducationalOccupationalProgram)

Type number

The maximum number of students who may be enrolled in the program.

financialAidEligible (inherited from EducationalOccupationalProgram)

Type string,DefinedTerm

A financial aid type or program which students may use to pay for tuition or fees associated with the program.

applicationDeadline (inherited from EducationalOccupationalProgram)

Type date

The date at which the program stops collecting applications for the next enrollment cycle.

occupationalCategory

Type CategoryCode,string

A category describing the job, preferably using a term from a taxonomy such as [BLS O*NET-SOC](http://www.onetcenter.org/taxonomy.html), [ISCO-08](https://www.ilo.org/public/english/bureau/stat/isco/isco08/) or similar, with the property repeated for each applicable value. Ideally the taxonomy should be identified, and both the textual label and formal code for the category should be provided.\n Note: for historical reasons, any textual label and formal code provided as a literal may be assumed to be from O*NET-SOC.
